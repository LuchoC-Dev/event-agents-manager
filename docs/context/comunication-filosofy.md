La aplicación NO está basada en chats.

Las comunicaciones NO son conversaciones libres.

Todo gira alrededor de instrucciones.

Una instrucción representa:
- una solicitud,
- una delegación,
- un pedido,
- una revisión,
- una aclaración,
- o cualquier intercambio organizacional estructurado.

Las instrucciones son entidades persistentes.

Cada instrucción:
- tiene lifecycle propio,
- estados propios,
- eventos propios,
- ownership propio,
- historial propio.

Las instrucciones NO son simples mensajes.

Una instrucción puede:
- ser recibida,
- aceptada,
- rechazada,
- respondida,
- completada,
- cancelada,
- invalidada,
- revisada,
- reenviada,
- o fallar.

Las instrucciones representan comunicaciones organizacionales persistentes.
