El sistema funciona dentro de proyectos.

Cada proyecto representa una organización completamente aislada.

No existe una organización global única.

Cada proyecto puede tener:
- agentes distintos,
- jerarquías distintas,
- relaciones distintas,
- workflows distintos,
- estructuras distintas.

La jerarquía organizacional existe, pero NO limita completamente las comunicaciones.

Ejemplo:
Frontend Lead
    ↓
Designer

Pero el Designer puede comunicarse lateralmente con:
- Research,
- QA,
- Backend,
- otros equipos.

Por lo tanto:
la estructura es híbrida:
- jerarquía organizacional,
- más colaboración lateral.

La jerarquía define:
- ownership,
- responsabilidad,
- delegación,
- supervisión,
- aprobación.

Las relaciones laterales representan:
- colaboración,
- consultas,
- revisiones,
- soporte,
- aclaraciones,
- investigación.

Las relaciones NO son permanentes.

Todo es dinámico:
- agentes,
- equipos,
- asignaciones,
- relaciones.

Los agentes pueden existir temporalmente.

El sistema NO debe asumir estructuras rígidas.
